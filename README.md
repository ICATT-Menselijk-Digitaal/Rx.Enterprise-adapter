# Rx.Enterprise-adapter

Adapter to retrieve information about people and businesses in KISS from Rx.Enterprise.

## Authentication

The Rx.Enterprise API uses mutual TLS (mTLS) — every request must include a client certificate issued per environment.

### Getting the certificates & running locally

The `.crt` and `.key` files are stored in **1Password**. Download both files and place them under:

```
src/RxEnterpriseZgwProxy/certs/client.crt
src/RxEnterpriseZgwProxy/certs/client.key
```

These paths are gitignored.

### Calling the API with curl

```bash
curl --cert certs/client.crt \
     --key  certs/client.key \
     -i 'https://rheden.connect.rx-enterprise-acc.nl/api/zaak/2025-000001'
```

### Running the worker locally

Copy the example env file and fill in the values:

```bash
cp src/RxEnterpriseZgwProxy/.env.example src/RxEnterpriseZgwProxy/.env
```

Then run:

```bash
dotnet run --project src/RxEnterpriseZgwProxy
```

## Rx.Enterprise to KISS/ZGW field mapping

The `RxEnterpriseZgwProxy` exposes the subset of ZGW-shaped endpoints that KISS calls. Rx.Enterprise remains the source system; this adapter maps the Rx response fields into the fields KISS reads.

### Zaak search and detail

KISS calls:

```text
GET /zaken/api/v1/zaken?identificatie=<zaaknummer>
GET /zaken/api/v1/zaken/{id}
```

The search call is translated to the Rx.Enterprise zaak search endpoint using the `identificatie` query value.

| KISS/ZGW field | Rx.Enterprise field | Notes |
| --- | --- | --- |
| `url` | generated by adapter | Points to `/zaken/api/v1/zaken/{sleutel}`. |
| `uuid` | `sleutel` | Used as the stable identifier in adapter responses. |
| `identificatie` | `sleutel` | Zaak number/id as known by Rx.Enterprise. |
| `omschrijving` | `betreft` | Main display text used by KISS. |
| `bronorganisatie` | none | Returned as an empty string. |
| `zaaktype` | `betreft` | |
| `registratiedatum` | `boekdatum` | |
| `startdatum` | `boekdatum` | Same source as `registratiedatum`. |
| `status` | none | will be revisited. |
| `toelichting` | `toelichting`, fallback `notitie`, fallback `omschrijving` | Returned on zaak detail as well as search results. |


### Status calls

```text
GET /zaken/api/v1/statussen/{id}
GET /api/zaken/statussen/{id}
```

The zaak response currently leaves `status` blank, so KISS should normally not follow a status URL from this adapter. The status endpoints exist as compatibility endpoints for callers that still request them.

| KISS/ZGW field | Source | Notes |
| --- | --- | --- |
| `url` | generated by adapter | Echoes the requested status endpoint. |
| `uuid` | encoded status id | URL-safe base64 representation of the status text/id. |
| `zaak` | none | Returned as `null`. |
| `statustype` | generated by adapter | Points to `/catalogi/api/v1/statustypen/{id}`. |
| `datumStatusGezet` | none | Returned as `null`. |
| `statustoelichting` | decoded status id | Only meaningful if a status value is supplied. |
| `indicatieLaatstGezetteStatus` | none | Returned as `null`. |

### Catalogi calls

```text
GET /catalogi/api/v1/zaaktypen/{id}
GET /catalogi/api/v1/statustypen/{id}
```

| Endpoint | Returned fields | Notes |
| --- | --- | --- |
| `/catalogi/api/v1/zaaktypen/{id}` | `onderwerp`, `omschrijving`, `doorlooptijd`, `servicenorm` | The id is decoded as URL-safe base64. `doorlooptijd` and `servicenorm` are `null`. |
| `/catalogi/api/v1/statustypen/{id}` | `omschrijving` | The id is decoded as URL-safe base64. |

### Documents and attachments

KISS retrieves zaak-document relations first, then follows the related enkelvoudiginformatieobject URL.

```text
GET /zaken/api/v1/zaakinformatieobjecten?zaak=<zaak-url>
GET /documenten/api/v1/enkelvoudiginformatieobjecten/{documentnummer}--{virtualid}
GET /documenten/api/v1/enkelvoudiginformatieobjecten/{documentnummer}--{virtualid}/download
```

For `zaakinformatieobjecten`, the adapter first gets the zaak from Rx.Enterprise, reads `documentnummer`, then gets the document metadata from Rx.Enterprise.

| KISS/ZGW field | Rx.Enterprise field | Notes |
| --- | --- | --- |
| `uuid` | derived from `{documentnummer}--{virtualid}` | Stable MD5-based UUID. |
| `informatieobject` | generated by adapter | Points to the corresponding enkelvoudiginformatieobject endpoint. |
| `zaak` | request query `zaak` | Echoes the zaak URL requested by KISS. |
| `registratiedatum` | `bijlagedatumtijd[virtualid - 1]` | Uses the parallel Rx attachment arrays. |
| `aardRelatieWeergave` | none | Fixed value: `Hoort bij, omgekeerd: kent`. |
| `titel`, `beschrijving` | none | Returned as empty strings. |
| `vernietigingsdatum`, `status` | none | Returned as `null`. |

For `enkelvoudiginformatieobjecten`:

| KISS/ZGW field | Rx.Enterprise field | Notes |
| --- | --- | --- |
| `identificatie` | `documentnummer` | Rx document number. |
| `bronorganisatie` | none | empty string. |
| `creatiedatum` | `creatiedatum` | From Rx document metadata. |
| `titel` | `bijlageomschrijving[virtualid - 1]`, fallback filename | |
| `vertrouwelijkheidaanduiding` | none | Fixed value: `openbaar`. |
| `formaat` | `bijlagecontenttype[virtualid - 1]` | Falls back to `application/octet-stream`. |
| `bestandsnaam` | `bijlageinfo[].filename` | Matched by `virtualid`. |
| `bestandsomvang` | `bijlagegrootte[virtualid - 1]` | Parsed as an integer; falls back to `0`. |
| `inhoud` | generated by adapter | Download URL for the attachment content. |
